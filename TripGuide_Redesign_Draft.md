# ร่างออกแบบ: คู่มือทริป (Trip Guide) — เวอร์ชันจัดหมวดหมู่ใหม่

> สถานะ: **ร่าง (Draft)** — ยังไม่เริ่มลงมือแก้โค้ด รอรีวิว/ปรับก่อน
> แนวทางที่เลือก: **"เล่าเป็นการเดินทาง"** — หมวด dynamic ไล่ตามเส้นทาง + การ์ด hero
> อ้างอิงทริปจริง: **กรุงเทพ → ชุมพร 2 วัน 1 คืน** (แวะหัวหิน/ประจวบระหว่างทาง)

---

## 1. ปัญหาของเวอร์ชันปัจจุบัน

ฝั่ง guest `TripGuide.jsx` มี 3 แท็บตายตัว: สถานที่ / ความรู้ทั่วไป / คลังศัพท์

ข้อจำกัด:

- หมวด hard-code ในโค้ด 2 จุด (`KNOWLEDGE_CATEGORIES`, `ARTICLE_CATEGORIES`) — staff เพิ่มเองไม่ได้
- แท็บ "ความรู้ทั่วไป" เป็นถังรวมทุกอย่าง หายากเมื่อเนื้อหาเยอะ
- การ์ดบทความแบนเท่ากันหมด ไม่มีลำดับความสำคัญ ไม่มี icon/สีบอกหมวด
- ไม่มีช่องค้นหาบทความ (มีเฉพาะ phrasebook)
- ทุกอย่างเป็น "สถานที่รายจุด" ไม่มีที่สำหรับเนื้อหาระดับทริป/จังหวัด/เส้นทาง

---

## 2. เป้าหมาย

1. หมวดบทความเป็น **dynamic** — staff เพิ่ม/ลบ/แก้ชื่อ/เรียง/เลือก icon+สี ได้เอง
2. จัดเนื้อหาให้ "เล่าเป็นการเดินทาง" ไล่จากภาพรวม → ระหว่างทาง → ปลายทาง → เตรียมตัว
3. การ์ดสวยขึ้น มี hero + icon/สีประจำหมวด + ค้นหาได้ทั้งคู่มือ
4. ไม่พังของเดิม — บทความ/หมวดปัจจุบัน migrate อัตโนมัติ

---

## 3. โครงหน้า Guest (โครงที่เลือก)

โครงบนสุดคง **2 โหมด**: **บทความ** และ **คลังศัพท์**

### โหมด "บทความ" — เรียงตามลำดับการเดินทาง

1. **เกี่ยวกับทริปนี้** — การ์ด **hero** ปักบนสุด (ภาพรวมทริป กี่วัน ไปไหน ธีม ต้อนรับ) มาจาก flag `is_featured`
2. **ไฮไลท์จังหวัด (ชุมพร)** — สปอตไลต์ปลายทาง: ของขึ้นชื่อ, เจ้าพ่อกรมหลวงชุมพรฯ, เอกลักษณ์ปักษ์ใต้ (ไม่ผูกจุดใดจุดหนึ่ง)
3. **ระหว่างทาง** — จุดบนเส้นทาง แบ่ง 2 ชนิด (ดู §4)
4. **จุดหมาย** — บทความรายสถานที่จริง (วัดสวี, หาดทรายรี ฯลฯ) ผูก `itinerary_item_id`
5. **เตรียมตัว & เกร็ดน่ารู้** — เนื้อหาปฏิบัติ/ทั่วไป (อากาศ, ของฝาก, การแต่งกายเข้าวัด/ศาล, จุดพักรถ) ปิดท้าย

> ลำดับ 1–5 คุมด้วย `sort_order` ของหมวด — staff จัดเองได้ ไม่ตายตัวในโค้ด

### โหมด "คลังศัพท์" — แยกต่างหาก

- ใช้ตาราง `phrasebook_entries` เดิม (ค้นหา + filter language pair + ตารางคำ/คำแปล/คำอ่าน) คงของเดิม
- ทริปในประเทศ (เช่นชุมพร) คำศัพท์แทบไม่จำเป็น → ควร **toggle ซ่อนทั้งโหมดได้** ต่อทริป

### เนื้อหา "ความรู้/วัฒนธรรม/เคล็ดลับ" เดิมไปไหน?

3 ถังตายตัวเดิมละลายเป็นหมวด dynamic โดยแยกตามธรรมชาติเนื้อหา:

- เนื้อหา "เกี่ยวกับที่เที่ยว" (ประวัติ/ตำนานของจุด) → เลื่อนขึ้นไปรวมใน **ไฮไลท์จังหวัด** หรือผูกกับ **จุดหมาย** นั้น
- เนื้อหา "ปฏิบัติ/ทั่วไป" → รวมเป็นหมวด **เตรียมตัว & เกร็ดน่ารู้** ท้ายสุด

ไม่มีอะไรตกหล่น — แค่ย้ายที่ตามบริบท

---

## 4. หมวด "ระหว่างทาง" — แวะ vs. ผ่าน

รวมทั้งจุดที่แวะจริง **และ** จุดน่าสนใจที่รถผ่าน (ไม่ได้แวะ) ในหมวดเดียว โดยใช้ `itinerary_item_id` (nullable — สคีมารองรับอยู่แล้ว) เป็นตัวแยก:

| ชนิด | เงื่อนไข | การแสดง | ลำดับ |
|---|---|---|---|
| **แวะจริง** | `itinerary_item_id` ไม่ null | ป้ายเขียว "แวะ · {เวลา}" | **แสดงก่อน** เรียงตามเวลา |
| **ผ่าน ไม่ได้แวะ** | `itinerary_item_id` = null | ป้ายเทา "ผ่าน · ไม่ได้แวะ" การ์ดสีจางลง | ต่อท้าย |

หลักการเนื้อหา:

- **ติดป้ายให้ชัดเสมอ** — กันลูกทัวร์เข้าใจผิดว่ารถจะจอด
- **คัดเฉพาะที่เห็นได้จริงจากรถ / มีเรื่องเล่า** (เพชรบุรี เขาวัง–ขนมหม้อแกง, หัวหิน, สวนมะพร้าวทับสะแก) — เป็น "ไกด์กระซิบข้างหู" ไม่ใช่วิกิพีเดียทุกอำเภอ

## 5. โครงสร้างข้อมูลใหม่ (Data model)

### 5.1 ตารางใหม่: `guide_categories`

| คอลัมน์ | ชนิด | หมายเหตุ |
|---|---|---|
| `id` | uuid (PK) | |
| `tour_id` | uuid (FK) | ผูกกับทัวร์ |
| `label_th` / `label_en` / `label_zh` | text | ชื่อหมวด 3 ภาษา |
| `icon` | text | ชื่อ icon จาก `Icon.jsx` |
| `color` | text | สีประจำหมวด (chip + แถบการ์ด) |
| `sort_order` | int | ลำดับหมวด (คุมลำดับการเดินทาง §3) |
| `is_active` | bool | ซ่อน/แสดงหมวด |

### 5.2 ปรับ `guide_articles`

- เพิ่ม `category_id` (FK → `guide_categories.id`)
- เพิ่ม `is_featured` (bool) — บทความ hero "เกี่ยวกับทริปนี้" ที่ปักบนสุด
- คง `category` (text) ชั่วคราวเพื่อ migrate
- คง `itinerary_item_id` (nullable) — ใช้แยกแวะ/ผ่านใน §4 ด้วย
- ที่เหลือเดิม: `title`, `body`, `source_url`, `image_url`, `sort_order`, `is_published`

### 5.3 phrasebook

ไม่แตะ — `phrasebook_entries` ใช้ `category` text อิสระเหมือนเดิม (คุมขอบเขตเฟสแรก)

### 5.4 Migration หมวดเดิม → หมวดใหม่

สร้างหมวดตั้งต้นอัตโนมัติแล้ว map บทความ 4 อันที่มีอยู่:

| เดิม (text) | ใหม่ (label_th) | icon แนะนำ |
|---|---|---|
| `place` | จุดหมาย | `map-pin` |
| `knowledge` | เตรียมตัว & เกร็ดน่ารู้ | `info` |
| `culture` | ไฮไลท์จังหวัด | `star` |
| `tips` | เตรียมตัว & เกร็ดน่ารู้ | `lightbulb` |

(staff ปรับ/เพิ่มหมวด เช่น "ระหว่างทาง", "อาหาร" ได้เองภายหลัง)

## 6. แนวทางเนื้อหา + รูป (ทริปชุมพร)

- **หมวดสถานที่/จุดหมาย ผูก 1:1 กับกำหนดการจริง** ผ่าน `itinerary_item_id` — ลูกทัวร์แตะจากหน้ากำหนดการเข้ามาอ่านได้
- จุดที่ควรมีบทความเต็ม: อุทยานราชภักดิ์, วัดสวี/พระบรมธาตุสวี, ชุมชนตลาดล่าง 100 ปี, ศาลกรมหลวงชุมพรฯ หาดทรายรี
- ที่พัก (A-TÉ Hotel) / จุดแวะกิน → การ์ดสั้นในหมวดที่เกี่ยว ไม่ต้องบทความเต็ม
- **รูป:** ภาพถ่ายจริง 1 รูป/บทความ hero แนวนอน 16:9 บีบอัด client (ตาม flow รูปกระเป๋าที่มีอยู่) เลี่ยง stock generic — ถ้าไม่มีรูปดี ใช้ icon ประจำหมวดแทน
- ไอเดียเนื้อหา ไฮไลท์/ความรู้: ประวัติกรมหลวงชุมพรฯ, ตำนานพระบรมธาตุสวี, ที่มา "ตลาดล่าง 100 ปี"
- เตรียมตัว: อากาศชุมพร, ของฝาก (กล้วยเล็บมือนาง กาแฟ), มารยาทเข้าวัด/ศาล, จุดพักรถ/ห้องน้ำ

---

## 7. งานฝั่ง Staff — GuideBuilder

- เพิ่มส่วน **"จัดการหมวดหมู่"**: ตารางหมวด (ชื่อ 3 ภาษา, icon, สี, เรียงลำดับ, toggle แสดง/ซ่อน, ลบ)
- ช่องเลือกหมวดตอนสร้าง/แก้บทความ → dropdown จาก `guide_categories` (แทน list ตายตัว)
- เพิ่ม toggle **"ตั้งเป็นบทความแนะนำ (hero)"** = `is_featured`
- กันลบหมวดที่ยังมีบทความ (เตือน/ย้ายก่อน)

จุดที่ต้องแก้ในโค้ด (คร่าวๆ):

- `pages/staff/GuideBuilder.jsx` — ลบ `ARTICLE_CATEGORIES`, เพิ่ม UI จัดการหมวด + toggle featured, โหลด categories
- `pages/guest/TripGuide.jsx` — ลบ `KNOWLEDGE_CATEGORIES`/`TABS` ตายตัว, เพิ่ม hero + ชิป/กลุ่มหมวด dynamic + ค้นหาบทความ + ป้ายแวะ/ผ่าน + การ์ดใหม่
- `locales/*.json` — ป้าย UI คงที่ (label หมวดย้ายไป DB)
- migration SQL: สร้างตาราง + `category_id` + `is_featured` + seed หมวด

---

## 8. ข้อควรตัดสินใจที่ยังเปิดอยู่

1. ป้ายหมวด 3 ภาษา (เสนอไว้) หรือภาษาเดียวพอ
2. layout ในหมวด: list อย่างเดียว หรือ grid 2 คอลัมน์เมื่อกรองหมวดเดียว
3. สี/ไอคอนหมวด: staff เลือกอิสระ หรือจำกัดชุด preset ให้ดูเป็นระบบ
4. โหมดคลังศัพท์: ซ่อนอัตโนมัติเมื่อไม่มีข้อมูล หรือให้ staff toggle เอง
